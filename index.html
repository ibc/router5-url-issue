router5 URL issue:

https://github.com/router5/router5/issues/248


You should be now in http://localhost:3000/Microsoft (so router5 matched the "company" route).

Open the browser console to see some logs. Also, window.ROUTER points to the router instance.

Now if you click on the following links, you should also remain in http://localhost:3000/Microsoft. However, due issue 248, router5 will not match some of the links below (3 and 4) and will move to the default route http://localhost:3000/default-route-company instead.


1) http://localhost:3000/Microsoft

2) http://localhost:3000/Microsoft/

3) http://localhost:3000/Microsoft?foo=123&bar=456

4) http://localhost:3000/Microsoft/?foo=123&bar=456

5) http://localhost:3000/Microsoft/login

6) http://localhost:3000/Microsoft/login/

7) http://localhost:3000/Microsoft/login?foo=123&bar=456

8) http://localhost:3000/Microsoft/login/?foo=123&bar=456


Ok, so links 3) and 4) did not work (they moved you to http://localhost:3000/default-route-company). Let's try 3) via API. Run in the console:

ROUTER.matchPath('/Microsoft?foo=123&bar=456')

Oh, it DOES match the "company" route, but it didn't when directly open the link 3) above!


Now let's try 4) via API:

ROUTER.matchPath('/Microsoft/?foo=123&bar=456')

It matches no route. No idea why not.
